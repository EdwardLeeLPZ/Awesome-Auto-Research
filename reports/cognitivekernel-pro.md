# CognitiveKernel-Pro (Tencent AI Lab)

> **An open-source deep research agent framework from Tencent AI Lab that achieves state-of-the-art performance on GAIA and other benchmarks using an SFT-only training recipe — outperforming RL-trained models like WebDancer and WebSailor.**

CognitiveKernel-Pro (CK-Pro) is a research framework from Tencent AI Lab that addresses a fundamental tension in the deep research agent field: existing capable agents rely on expensive reinforcement learning or large proprietary models, making them difficult to reproduce and improve upon. CK-Pro demonstrates that a carefully constructed supervised fine-tuning (SFT) dataset, derived from agent trajectories produced by CK-Pro itself, is sufficient to train an open-weight Qwen3-8B model that outperforms RL-trained systems such as WebDancer and WebSailor on standard benchmarks.

The system pairs this trained model with a multi-agent scaffold that includes a Playwright-powered web browser engine, a file-processing agent, and a coordinating main agent. The full pipeline can operate using only free tools (DuckDuckGo search) apart from the web browser infrastructure, which is self-hosted.

The architecture is a three-agent hierarchy: a main coordinating agent that reasons about the task and delegates subtasks, a web agent that handles all browser-based interactions, and a file agent that processes local documents of any format. All three share a unified LLM backend which can be swapped between the trained Qwen3 model and commercial APIs.

## Core Workflow

### Step 1 — Task Receipt and Planning

The main CKAgent receives a task (e.g., a GAIA question or an open-domain research query) and produces an initial plan. Planning is implicit in the agent's chain-of-thought — there is no separate planner module.

### Step 2 — Web Search Delegation

When the task requires web information, the main agent delegates to the WebAgent. The WebAgent:
1. Launches (or re-uses) a Playwright-backed Node.js web browser service listening on a configurable port.
2. Submits queries to Google Search (via API) or DuckDuckGo (free).
3. Navigates to result pages and returns either raw text or a screenshot (depending on whether a VLM is needed for visual content).

### Step 3 — File Processing Delegation

For tasks involving documents, the FileAgent handles ingestion:
- PDF: `pdfminer-six` for text extraction; `pdf2image` for scanned pages.
- Office files: `python-pptx`, `mammoth` (DOCX), `libreoffice` for conversion.
- Audio/Video: `ffmpeg` + `SpeechRecognition` for transcript extraction.
- Multimodal pages: a VLM (GPT-4.1 vision or equivalent) processes screenshots.

### Step 4 — Iterative Reasoning and Tool Use

The main agent iterates over tool calls, accumulating context across multiple web and file interactions. The underlying trained model is specifically fine-tuned to produce high-quality multi-turn reasoning trajectories, including appropriate tool-call and response patterns.

### Step 5 — Answer Synthesis

After accumulating sufficient evidence, the main agent produces a final answer or report. For GAIA-style tasks this is a structured final answer; for open-domain queries it is a prose synthesis with source attribution.

## Key Features

- **SFT-only training recipe** — reproduces or exceeds the performance of RL-trained deep research agents (WebDancer, WebSailor) using only supervised fine-tuning, making the training process more stable and reproducible.
- **Open-weight trained model** — Qwen3-8B-CK-Pro is freely downloadable from HuggingFace, enabling fully local inference with no commercial API dependency.
- **Self-hosted Playwright web engine** — a Node.js server provides full JavaScript-rendering web access, eliminating the need for expensive external browser APIs.
- **Multimodal file support** — the FileAgent handles PDF, PPTX, DOCX, audio, and video inputs, making the system capable on multi-modal GAIA tasks.
- **Provider-agnostic main agent** — the main LLM can be swapped to GPT-4.1, Claude, or any vLLM-compatible endpoint with a single configuration change.
- **Free search fallback** — DuckDuckGo can replace the Google Search API, enabling zero-cost web access.
- **Fully open dataset** — the SFT training data is released publicly on HuggingFace for community fine-tuning.
- **Sandboxed execution** — the documentation explicitly recommends and shows how to run the system with reduced OS privileges, as agent-generated code is executed directly.

## Technical Implementation

### Model Training Pipeline

CK-Pro introduces a data-generation-then-SFT cycle:

1. **Trajectory Collection** — The CK-Pro agent (with a strong teacher model) solves a curated query set, recording the full multi-turn tool-call trajectory.
2. **Quality Filtering** — Trajectories are filtered for correct final answers, valid tool-call/response pairing, and token count bounds.
3. **Masked Trajectory Guidance (MTG)** — An LLM-powered rewriting pass improves trajectory quality by correcting sub-optimal reasoning steps while preserving the overall structure.
4. **SFT Training** — The cleaned trajectories are used to fine-tune Qwen3-8B using standard instruction-tuning protocols.

The companion technical report [arXiv:2510.14438](https://arxiv.org/abs/2510.14438) (*Explore to Evolve: Scaling Evolved Aggregation Logic via Proactive Online Exploration*) extends this with online exploration to further improve the aggregation logic.

### Web Engine Architecture

The Playwright web engine is a Node.js HTTP server that accepts navigation and extraction commands from the Python agent via HTTP. It provides:
- Page loading with full JavaScript execution.
- DOM-to-text extraction for standard pages.
- Screenshot generation for visual or JS-heavy pages (consumed by VLM).
- Session persistence (optional) for multi-step web interactions.

This design decouples the browser environment from the Python agent, allowing them to run on different machines or in different security contexts.

### Configuration System

All agent parameters are specified as a nested Python dictionary passed via `--updates`. This `KwargsInitializable` pattern allows hierarchical overrides:

```python
{
  "model": {"call_target": "gpt:gpt-4.1"},
  "max_steps": 10,
  "web_agent": {
    "model": {"call_target": "http://vllm-server:8080/v1/chat/completions"},
    "web_env_kwargs": {"web_ip": "localhost:3000"}
  }
}
```

### Evaluation on GAIA

CK-Pro reports results on the GAIA benchmark (level 1/2/3), which tests general AI assistant capability on complex multi-step tasks. The system is evaluated in both *agentic mode* (real web access) and *closed mode* (local context only), with the trained Qwen3-8B-CK-Pro model producing competitive results against GPT-4-level commercial baselines.

## Strengths

1. **SFT outperforms RL at the same scale** — the empirical demonstration that carefully curated SFT data beats RL for deep research agents is a significant contribution; it simplifies the training pipeline and improves reproducibility.

2. **Fully open training stack** — model weights, SFT dataset, and training methodology are all published, enabling the community to reproduce, audit, and extend the results.

3. **Comprehensive multimodal support** — the FileAgent handles a wider range of input formats (audio, video, scanned PDFs) than most comparable systems, making CK-Pro competitive on the full GAIA benchmark suite.

4. **Self-hosted browser engine** — running Playwright locally avoids per-call API costs for web access and allows the system to access intranet or paywalled resources in appropriate settings.

5. **Production-quality sandboxing guidance** — the documentation provides specific `sudoers` and hostname isolation commands, reflecting genuine security thinking for code-executing agents.

6. **Modular LLM backend** — the three-agent system (main, web, file) can use different LLMs for each role, allowing cost/capability trade-offs (e.g., cheap model for file extraction, powerful model for synthesis).

---

## Limitations

1. **Complex setup** — the system requires a separately running Node.js web browser service, a vLLM server (or API key), and multiple Python packages including system-level dependencies (`libreoffice`, `ffmpeg`, `poppler`). The setup burden is significantly higher than simple API-based agents.

2. **No memory or persistence** — like most deep research agents, CK-Pro does not maintain state between separate task invocations. Each run starts from scratch.

3. **Sandboxing is advisory, not enforced** — agent-generated code is executed directly in the host Python process; the recommended `sudoers` restriction is a mitigation, not a full sandbox. Enterprise deployments would need a proper container or VM boundary.

4. **English-centric evaluation** — GAIA is primarily English. Performance on multilingual research tasks has not been systematically evaluated.

5. **Small model scale (8B)** — the flagship trained model is Qwen3-8B. For tasks requiring deep domain reasoning or long-context synthesis, this scale may be limiting compared to 70B+ models.

6. **Proprietary license** — unlike MIT-licensed alternatives, the CK-Pro code is under Tencent's proprietary terms. This limits commercial use and contributions from the broader community.
