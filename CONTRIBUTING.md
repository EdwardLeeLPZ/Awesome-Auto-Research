# Contributing to Awesome-Auto-Research

Thank you for your interest in contributing! This repository is a curated list of autonomous research systems and agents. We welcome contributions that add value to the community.

## 📋 How to Contribute

### 1. Adding a New Project to the Main List

To add a project to the main `README.md` table, please follow these steps:

1. **Fork** this repository.
2. **Add your project** to the appropriate category table in `README.md`.
3. Use the standard row format:
   ```
   | [Project Name](link) | ![Stars](shield-badge-url) | Framework | LLM Support | Short description |
   ```
4. **Submit a Pull Request** with the title: `Add: [Project Name]`

### 2. Adding a New Detailed Report

We especially encourage contributions that add **in-depth analysis reports** to the `reports/` folder.

Each report should follow the standard template in [`reports/README.md`](reports/README.md).

Steps:
1. Fork this repository.
2. Create a new file `reports/<project-slug>.md` (e.g., `reports/my-research-agent.md`).
3. Fill in **all sections** of the report template.
4. Add an entry to `reports/README.md` index.
5. Optionally link the report from the relevant section in the main `README.md`.
6. Submit a Pull Request with title: `Report: [Project Name]`

### 3. Fixing Errors or Outdated Information

If you spot outdated links, wrong descriptions, or factual errors:
- Open an **Issue** first to discuss the change.
- Or directly submit a small PR with a clear explanation.

---

## ✅ Quality Standards

- **No spam or self-promotion without merit.** Projects must have clear relevance to autonomous research.
- **Links must be valid.** Check that all URLs are accessible.
- **Stars requirement (recommended):** ≥ 100 GitHub stars for main-list inclusion (exceptions for very recent, high-impact work).
- **Reports must be complete.** All required sections of the report template must be filled.
- **Objectivity.** Reports should be balanced and factual. Avoid marketing language.
